Day 2 · 정정 — 실제 CE 매니페스트 확인

nginx-ingress 가정 폐기 → askbot 패턴

CORRECTION 추측으로 채우지 않으려고 moreh-dev/ce-training/server/k8s(실제 ce-blog·askbot 매니페스트)를 직접 읽음. 결과: 내 nginx-ingress + cert-manager + 레지스트리 가정이 전부 틀림.

  • TLS는 클러스터 밖 public edge(1.249.213.127)에서 종료 → cert-manager 불필요.
  • 공개는 istio 공유 게이트웨이 istio-system/public-gw + HTTPRoute(Gateway API) → Ingress 불필요.
  • 커스텀 이미지를 레지스트리에 푸시하지 않음. askbot은 노드 systemd 호스트 서비스로 돌고, 클러스터엔 selector-less Service + EndpointSlice → 노드 IP(192.168.2.20=ce-master).
  • askbot은 XHR 위젯이라 구글 oauth2-proxy를 안 씀 → 앱 자체 게이트(ACCESS_TOKEN+CORS).

DECISION Repo Lens는 정적 블로그가 아니라 askbot과 같은 XHR + 자체 GitHub OAuth + AUTH_REQUIRED 앱 → askbot 패턴 그대로. 구글 게이트를 앞에 두면 /api XHR과 /gh/callback이 깨짐.

재작성: deploy/k8s/repolens.yaml(ns repo-lens + selector-less Svc + EndpointSlice + HTTPRoute, X-Forwarded-Proto: https·timeouts 0s), deploy/repolens.service(ce-askbot.service 미러), deploy/k8s/apply.sh, 런북 docs/repo-lens-ce-deploy.html 전면 개정. placeholder 3개(NAMESPACE/REGISTRY/CLUSTER_ISSUER) 전부 해소 — 남은 호스트값은 노드 IP·DNS(엣지 와일드카드면 자동)뿐.
