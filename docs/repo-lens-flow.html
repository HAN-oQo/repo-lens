① 즉시 — POST /api/repo

응답이 오면 트리·README가 바로 보이고 화면이 인터랙티브해집니다. (프론트: app/page.tsx loadRepo())

  1. URL 파싱·검증 — parseRepo() server/lib/repo.mjs. github.com owner/repo만 허용(SSRF 차단).
  2. Clone / 업데이트 — ensureClone(): 
    # 최초 (얕은 클론, DATA_DIR/repos/<owner>/<repo>)
git clone --depth 1 --single-branch [--branch <ref>] <url> <dir>
# 재방문(이미 있으면) → 빠른 최신화
git fetch --depth 1 origin <ref>  &&  git reset --hard FETCH_HEAD
git rev-parse HEAD                 # sha 확정
    토큰은 argv가 아니라 git extraheader(env)로 주입 — 로그/프로세스 목록에 안 남음.
  3. 파일 목록 — listTree() = git ls-files -z → 평면 리스트.
  4. README 선택·읽기 — findReadme() → readRepoFile() (2MB 캡).
  5. 그래프 예약 — requestGraph(owner,repo,dir) 호출 후 await 안 함 → ②가 백그라운드로 시작.
  6. 응답 {repo, tree, readmePath, readme, graph} → 프론트가 buildTree()로 트리 구성, README 탭 오픈.

② 백그라운드 — 심볼 그래프 graphify

①과 동시에, 응답을 막지 않고 진행. (server/lib/graph.mjs → server/lib/graphify.mjs)

  1. 동시 빌드 2개로 제한(큐). 상태 building.
  2. graphify update .      # tree-sitter 심볼 그래프 + Leiden 커뮤니티, LLM 미사용(무료·로컬)
# → graphify-out/graph.json (NetworkX node-link)
  3. toGraphData() 로 변환: community→색 group, source_file→클릭 점프, relation/confidence→엣지, in/out-degree로 허브. 상태 ready로 캐시.
그래프가 무겁기 때문에 백그라운드입니다. Ask는 그래프가 없어도 ripgrep만으로 먼저 동작하고, 그래프가 준비되면 이웃 확장으로 더 정확해집니다.

③ 온디맨드 — 클릭할 때마다

AskPOST /api/ask아래 GraphRAG server/lib/graphrag.mjs

Ask = GraphRAG

  1. 검색 relevantFiles: 질문 키워드로 ripgrep 히트 + (그래프 ready면) 그래프 1-hop 이웃 확장 → 관련 파일 선별.
  2. 컨텍스트 buildContext: 그 파일 슬라이스를 토큰 예산 안에서 모음(파일당 6000자/총 40000자 캡) → 출처 목록.
  3. LLM callLLM: ANTHROPIC_API_KEY(클라우드 모델) 또는 askbot(POST /ask→{id}→/result 폴링, x-access-token+origin). askbot 토큰은 CE에서 /srv/askbot/askbot.token 자동 로드.
  4. {answer, sources} 반환 → 답변 + 클릭 가능한 파일 출처.

설계 포인트

  • 클론·그래프는 서버에서 1회 + 디스크 캐시 → 재방문/파일 열기/검색이 즉시. 대용량도 트리·검색·파일은 초 단위.
  • 무거운 건 백그라운드 → UI 안 막힘. 진행 상황은 활동 로그(/api/activity)로 실시간: cloning → scanned → graph building → graph ready → search → ask.
  • 모델 선택은 /api/models가 askbot /models에서 라이브 디스커버리(클라우드 + 로컬 게이트웨이) → Ask 드롭다운에 채움.
  • CORS: /api/*는 Origin을 반사 — 공개 데모(Pages→CE)·로컬(127.0.0.1/포트 차이) 모두 교차 오리진이라 필요. 게이트는 GitHub 토큰/ AUTH_REQUIRED가 담당.
