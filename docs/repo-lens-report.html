1 무엇인가

링크 하나를 넣으면 왼쪽에 디렉토리 트리, 가운데에 README와 코드(Monaco = VSCode 에디터 본체)와 놀리지 그래프(파일 간 import 의존성), 오른쪽에 레포에 grounding된 AI Ask 패널이 뜹니다. 백엔드는 기본적으로 없고(정적), 사내 배포에서만 로그인용 작은 서버 하나가 붙습니다.

영역구현
파일 트리 / 코드 / READMEGitHub git-tree(재귀 1콜) + raw.githubusercontent.com, Monaco, react-markdown
놀리지 그래프JS/TS·Python import를 브라우저에서 정규식 추출·해석 → react-force-graph
AI Ask블로그 ask.js 멀티프로바이더 이식(기본 askbot, 또는 BYO 키), 레포 컨텍스트로 grounding
private 레포“Sign in with GitHub”(OAuth) 또는 토큰 붙여넣기 — 토큰은 브라우저 localStorage에만

2 시스템 토폴로지

아래는 사내 배포(repolens.ce.moreh.dev) 기준 구성도입니다. 정적 앱과 OAuth 엔드포인트가 같은 컨테이너·같은 origin에 있고, 데이터(GitHub)와 AI(askbot)는 브라우저가 직접 호출합니다.

구성요소역할비고
Cloudflare DNSmoreh.dev 존의 호스트명 → IP 해석DNS-only(회색) 또는 프록시(주황)
CE ingressTLS 종단 + 호스트명 라우팅cert-manager/Let’s Encrypt 추정
repo-lens container정적 ./out 서빙 + /gh/* OAuthserver/server.mjs, :8080
api.github.com / raw레포 메타·트리·파일브라우저가 토큰으로 직접
github.com OAuth로그인 동의 + code→tokensecret은 컨테이너에만
askbot.ce.moreh.devAI 응답origin 허용목록 게이트

공개 데모는 동일한 정적 앱을 GitHub Pages(han-oqo.github.io/repo-lens)로 서빙합니다. 여기엔 OAuth 서버가 없어 private 레포는 토큰 붙여넣기로만 봅니다.

3 ce.moreh.dev의 원리 (DNS)

공개 DNS를 조회해 확인한 사실입니다. moreh.dev의 네임서버는 Cloudflare이고, askbot.ce.moreh.dev는 A 레코드로 회사 엣지 공인 IP를 직접 가리킵니다(프록시 IP가 아니라 실 IP → DNS-only). ce는 존 위임이 아니라 단순 서브도메인 네이밍 관례입니다.

# 직접 확인
$ dig +short NS moreh.dev
dion.ns.cloudflare.com.
sky.ns.cloudflare.com.

$ dig +short A askbot.ce.moreh.dev
<CE 엣지 공인 IP>            # Cloudflare 프록시 IP가 아님 = DNS-only

그래서 새 서비스 추가 = 한 줄. Cloudflare에 repolens.ce.moreh.dev A(또는 인그레스로 CNAME)를 추가하고, CE 인그레스에 그 호스트 라우트 + TLS 인증서를 붙이면 끝. askbot과 똑같은 패턴입니다.

4 로그인 흐름 (OAuth)

정적 페이지는 client_secret을 안전하게 못 들고, GitHub은 PKCE도 토큰 엔드포인트 CORS도 지원하지 않습니다. 그래서 secret을 쥐고 code→token 교환만 하는 작은 서버(server/server.mjs)가 필요합니다. 토큰은 URL fragment로 앱에 전달되어 서버 로그에 남지 않습니다.

워커가 아니라 사내 한 박스로 갈 때는 같은 로직을 server/server.mjs가 처리합니다. 워커 버전은 workers/github-oauth.worker.js로 동봉.

5 데이터 흐름 & 레이트리밋

레포 1개 로드의 GitHub API 호출은 딱 3번이고, 파일 내용은 API 한도를 안 먹는 raw CDN에서 가져옵니다.

단계호출API 소모
레포 메타GET /repos/:o/:r1
파일 트리GET /repos/:o/:r/git/trees/:branch?recursive=11
언어 비율GET /repos/:o/:r/languages1
파일 내용 (public)raw.githubusercontent.com/...0
파일 내용 (private)contents API (+ >1MB는 blobs API)토큰 한도(5000/h)

미인증 60/h로도 public 레포는 충분하고, 토큰을 넣으면 5000/h + private 접근. 받은 파일은 IndexedDB에 캐시됩니다.

6 배포 ① 공개 데모 (GitHub Pages)

main에 push하면 .github/workflows/pages.yml가 정적 빌드(basePath /repo-lens) 후 Pages에 배포합니다.

# 레포는 HAN-oQo 소유 — 기본 활성 계정(hanq-moreh)에서 전환
gh auth switch --user HAN-oQo
git add -A && git commit -m "..."
git push origin main                 # → 워크플로우 트리거
gh run watch --repo HAN-oQo/repo-lens
gh auth switch --user hanq-moreh     # 원래 계정 복구

최초 1회 Pages 활성화: gh api -X POST repos/HAN-oQo/repo-lens/pages -f build_type=workflow

7 배포 ② 사내 (repolens.ce.moreh.dev)

사람이 한 번 준비할 것: OAuth App(client_id/secret, 콜백 https://repolens.ce.moreh.dev/gh/callback), .env, Cloudflare DNS + CE 인그레스/TLS, 그리고 askbot CORS 허용목록에 새 origin 추가.

# 1) OAuth App 등록 (GitHub → Settings → Developer settings → OAuth Apps)
#    Homepage:  https://repolens.ce.moreh.dev
#    Callback:  https://repolens.ce.moreh.dev/gh/callback

# 2) 시크릿/설정
cp .env.example .env        # GH_CLIENT_ID / GH_CLIENT_SECRET 채우기

# 3) 컨테이너 빌드 (basePath=루트, OAuth base 베이크)
scripts/repolens.sh image

# 4) 로컬 스모크 → CE 레지스트리 push → 인그레스 뒤 Deployment 롤
scripts/repolens.sh run
docker tag repolens:latest <registry>/repolens:latest && docker push <registry>/repolens:latest

Docker 없이 로컬 확인: scripts/repolens.sh build-ce && scripts/repolens.sh serve. 상세는 DEPLOY.md.

사내 전용 유지: 인그레스 소스 IP 허용목록/VPN, 또는 Cloudflare Access(Zero Trust). private 소스 + 토큰을 다루므로 공개 IP 노출 금지.

8 처음부터 재현

빈 디렉토리에서 전체를 다시 만드는 명령 순서입니다.

# 스캐폴드 + 의존성
npx create-next-app@latest . --ts --app --no-src-dir --no-tailwind \
    --no-eslint --import-alias "@/*" --use-npm --no-turbopack
npm install --legacy-peer-deps @monaco-editor/react react-markdown \
    remark-gfm rehype-highlight highlight.js react-force-graph-2d idb

# 개발 / 빌드
npm run dev                  # localhost:3000
npm run build                # 정적 export → ./out

# 레포 생성 + 배포 (HAN-oQo)
git init -b main && git add -A && git commit -m "init"
gh auth switch --user HAN-oQo
gh repo create HAN-oQo/repo-lens --public --source=. --remote=origin --push
gh api -X POST repos/HAN-oQo/repo-lens/pages -f build_type=workflow

핵심 파일: app/page.tsx(상태), lib/github.ts(fetch+OAuth), lib/imports.ts(그래프), components/AskPanel.tsx(AI), server/server.mjs(self-host).

9 보안

수정됨 — 오픈 리다이렉트(높음). redirect 허용 검사가 bare startsWith(prefix)여서 app.example.com.evil.com이 통과, 토큰 탈취 가능했음. r===p || startsWith(p+"/")로 수정(서버·워커 모두). 공격→400, 정상→302 검증.
강화 — 경로 탈출. 정적 핸들러가 정규화 후 resolved 경로가 OUT_DIR 안인지 확인(밖이면 403). /../../etc/passwd 탈출 불가 확인.
  • XSS 표면 — AI 답변은 lib/md.ts가 먼저 escape 후 화이트리스트 태그/링크만 생성. README는 react-markdown(raw HTML·javascript: 차단). eval 없음.
  • 토큰 — OAuth/PAT 토큰은 localStorage에만, fragment로 받은 뒤 history.replaceState로 제거. secret은 서버에만.
  • 스코프 — OAuth App은 repo read:org(읽기전용 repo 스코프 부재). 최소권한은 GitHub App(per-repo Contents/Metadata read)으로 가능.
  • 알려진 — npm audit가 Next가 끌어오는 postcss 1건(신뢰 못할 CSS 빌드 시에만 해당, 우리 영향 미미). Dependabot 추적.

지속 점검(꾸준히): .github/dependabot.yml(주간 npm·actions PR) + 배포 전 scripts/repolens.sh audit / /security-review. 기록은 SECURITY.md.
