Repo Lens — CE 클러스터 수동 배포

askbot과 동일한 방식으로 repolens.ce.moreh.dev 공개. 본인 인프라(무료) · GitHub 로그인 게이트 · 읽기 전용.

패턴: 호스트 systemd 서비스 + istio HTTPRoute · 근거: moreh-dev/ce-training server/k8s · 갱신 2026-06-17

웹 = 브라우저 클릭 · CLI = 터미널(ce-master에서). «…» 는 본인 값.

왜 이 모양인가: ce-blog의 정적 사이트는 구글 oauth2-proxy 뒤에 있지만, Repo Lens는 askbot처럼 XHR(/api/*) + 자체 GitHub OAuth(/gh/*) 앱입니다. 구글 인터랙티브 게이트를 앞에 두면 XHR과 콜백이 깨집니다. 그래서 askbot과 똑같이 — 호스트 서비스 + selector-less Service/EndpointSlice + 공유 게이트웨이 HTTPRoute로 공개하고, 게이팅은 앱 자체(GitHub 로그인 + AUTH_REQUIRED=1)가 합니다. 그래서 레지스트리·cert-manager·ingress가 필요 없습니다(TLS는 엣지에서 종료).
public edge 1.249.213.127 ← *.ce.moreh.dev TLS 종료 (SRE 관리, 기존) │ http (Host: repolens.ce.moreh.dev) ▼ istio public-gw :80 ← istio-system/public-gw (공유 Gateway API) │ HTTPRoute repolens (X-Forwarded-Proto: https 강제, request timeout 0s) ▼ repolens-origin Service(selector 없음) ─► EndpointSlice ─► ce-master 192.168.2.20:8080 └─ 호스트 systemd 서비스 (deploy/repolens.service) clone · ripgrep · graphify · GraphRAG 앱 자체 게이트: GitHub 로그인 + AUTH_REQUIRED=1 (askbot의 ACCESS_TOKEN+CORS에 해당)

1 호스트 서비스 띄우기 (실제 배포) CLI

ce-master에서 이미지를 로컬 빌드(푸시 없음)하고 systemd로 상시 실행. askbot의 ce-askbot.service와 같은 방식입니다.

# 0) 디렉토리 + 코드
sudo mkdir -p /srv/repolens/data && sudo chown -R $USER /srv/repolens
git clone https://github.com/HAN-oQo/repo-lens /srv/repolens/repo
cd /srv/repolens/repo

# 1) 이미지 로컬 빌드 (호스트명이 빌드 시점에 박힘 → build-arg)
docker build -t repolens:latest \
  --build-arg NEXT_PUBLIC_API_BASE=https://repolens.ce.moreh.dev \
  --build-arg NEXT_PUBLIC_OAUTH_BASE=https://repolens.ce.moreh.dev .
# (scripts/repolens.sh image 도 동일)

# 2) 환경파일 (chmod 600, git에 넣지 말 것) — .env.example 참고
cat > /srv/repolens/repolens.env <<'EOF'
PORT=8080
HOST=0.0.0.0
DATA_DIR=/data
AUTH_REQUIRED=1
ALLOWED_REDIRECTS=https://repolens.ce.moreh.dev
CLONE_ALLOWED_HOSTS=github.com
GH_CLIENT_ID=«oauth client id»
GH_CLIENT_SECRET=«oauth client secret»
# LLM 둘 중 하나: 사내 askbot 재사용(게이트됨 — 토큰 필수) 또는 Anthropic 직통
ASK_URL=https://askbot.ce.moreh.dev/ask
ASK_TOKEN=«/srv/askbot/askbot.token 값»   # 없으면 askbot이 {"error":"unauthorized"}
ASK_MODEL=claude-moreh-Qwen3.6-27B        # askbot 게이트웨이가 서빙하는 모델
# ANTHROPIC_API_KEY=«key»                 # 직통 쓰면 위 ASK_* 대신 이것만
EOF
chmod 600 /srv/repolens/repolens.env

# 3) systemd 서비스 등록
sudo cp deploy/repolens.service /etc/systemd/system/
sudo systemctl daemon-reload && sudo systemctl enable --now repolens
journalctl -u repolens -f          # 기동 로그
curl -s localhost:8080/healthz     # → ok
엄격한 비공개(SSH 터널)만 원하면 여기서 끝. repolens.service 의 바인드를 127.0.0.1:8080 으로 바꾸고 2번(HTTPRoute)을 적용하지 않으면 됩니다 — ssh -L 8080:localhost:8080 ce-master 로만 접근. 공개 게이트(아래 2번)는 선택입니다.

2 공개 라우트 적용 CLI

클러스터엔 Service + EndpointSlice + HTTPRoute만 — 워크로드는 호스트에 있습니다. (deploy/k8s/repolens.yaml)

cd /srv/repolens/repo
deploy/k8s/apply.sh
# 또는 직접:
export KUBECONFIG=/etc/kubernetes/admin.conf
sudo -E kubectl apply -f deploy/k8s/repolens.yaml   # HTTPRoute가 마지막 = 공개되는 순간
매니페스트의 노드 IP는 192.168.2.20(askbot과 동일한 ce-master). 다르면 kubectl get node -o wide 로 확인 후 EndpointSlice의 addresses 수정.

3 확인 CLI + 웹

# 호스트 로컬
curl -s localhost:8080/healthz                                   # → ok
# 라우트/엔드포인트
sudo -E kubectl -n repo-lens get httproute repolens
sudo -E kubectl -n repo-lens get endpointslice repolens-origin-1 -o wide
# 공개 경로 — 게이트웨이 통과 헬스
curl -s https://repolens.ce.moreh.dev/healthz                    # → ok
# 미인증 API는 차단(게이트 동작 확인)
curl -s -o /dev/null -w '%{http_code}\n' "https://repolens.ce.moreh.dev/api/status?repo=x"  # → 401

웹 https://repolens.ce.moreh.dev 열기 → Sign in with GitHub → 레포 Load → 트리·전문검색·심볼 그래프·GraphRAG.

4 (선택) 공개 데모도 이 백엔드로 CLI

han-oqo.github.io/repo-lens 가 이 백엔드를 쓰게 하려면 repo 변수만 설정(설정 전엔 브라우저 전용 v1 유지):

gh auth switch --user HAN-oQo
gh variable set REPOLENS_API_BASE   --repo HAN-oQo/repo-lens --body https://repolens.ce.moreh.dev
gh variable set REPOLENS_OAUTH_BASE --repo HAN-oQo/repo-lens --body https://repolens.ce.moreh.dev
gh workflow run pages.yml --repo HAN-oQo/repo-lens
gh auth switch --user hanq-moreh

그리고 OAuth App 콜백에 https://repolens.ce.moreh.dev/gh/callback 가 이미 있으니, ALLOWED_REDIRECTS에 https://han-oqo.github.io 도 추가하면 공개 데모 로그인 → v2 동작.

5 업데이트 · 롤백 CLI

6 트러블슈팅

증상원인 / 해결
공개 URL 404 / 응답 없음HTTPRoute가 public-gw 에 attach 됐는지(kubectl -n repo-lens describe httproute repolens → Parents의 Accepted=True). 게이트웨이가 네임스페이스를 라벨로 제한하면 repo-lens ns에 동일 라벨 추가. 호스트명·DNS(엣지) 확인.
502 / endpoint not ready호스트 서비스가 떠 있나(systemctl status repolens, curl localhost:8080/healthz). EndpointSlice의 노드 IP가 맞나(kubectl get node -o wide).
로그인 후 http로 리다이렉트/쿠키 문제HTTPRoute의 X-Forwarded-Proto: https 필터 적용 확인(엣지에서 TLS 종료되므로 필수). 콜백 URL이 https://repolens.ce.moreh.dev/gh/callback 와 정확히 일치하는지.
로그인 후 401 반복ALLOWED_REDIRECTS ≠ 실제 오리진, 또는 OAuth Client ID/secret 오타. journalctl -u repolens 확인.
private aiandlabs 레포 404OAuth App을 org(SSO)에서 Authorize 안 함 → org 설정에서 승인.
Ask가 느리다 504HTTPRoute timeouts.request: "0s" 적용 확인(LLM 호출이 게이트웨이 기본 타임아웃을 넘김).
Ask "No LLM configured"repolens.env 의 ASK_URL 또는 ANTHROPIC_API_KEY 설정. askbot 재사용 시 클러스터에서 askbot.ce.moreh.dev 도달 가능한지.
