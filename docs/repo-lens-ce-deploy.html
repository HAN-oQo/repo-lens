Repo Lens — CE 클러스터 수동 배포 (nginx)

ce-blog와 같은 방식으로 repolens.ce.moreh.dev에 배포. 본인 인프라(무료) · 로그인 게이트 · 읽기 전용.

ingress: nginx · manifest: deploy/k8s/repolens.yaml · 작성 2026-06-17

웹 = 브라우저 클릭 작업 · CLI = 터미널 명령. «…» 는 본인 값으로 치환.

왜 안전한가: 호스트는 닿을 수 있어도 AUTH_REQUIRED=1 이라 GitHub 로그인한 사용자만 API를 씁니다. 게다가 코드 읽기 전용 뷰어이고, GitHub 토큰은 브라우저에만, clone은 github.com만 허용합니다.

1 이미지 빌드 & 푸시 CLI

NEXT_PUBLIC_* 는 빌드 시점에 박힙니다 → 호스트명을 build-arg로 전달.

cd /path/to/repo-lens
docker build \
  --build-arg NEXT_PUBLIC_API_BASE=https://repolens.ce.moreh.dev \
  --build-arg NEXT_PUBLIC_OAUTH_BASE=https://repolens.ce.moreh.dev \
  -t «REGISTRY»/repolens:latest .
docker push «REGISTRY»/repolens:latest
빌드 머신 ≠ 클러스터 노드여도 됩니다. 멀티아키(예: arm 맥에서 빌드→amd64 노드)면 docker buildx build --platform linux/amd64 … 로.

2 매니페스트 값 채우기 CLI

deploy/k8s/repolens.yaml 의 «…» 를 ce-blog와 동일하게:

placeholder값
«NAMESPACE»ce-blog/askbot와 같은 네임스페이스
«REGISTRY»1번에서 푸시한 이미지 경로
«CLUSTER_ISSUER»cert-manager ClusterIssuer 이름(예: letsencrypt-prod)
Secret 값OAuth Client ID/secret, (선택) ANTHROPIC_API_KEY

ingressClassName은 이미 nginx, 타임아웃/인증서 어노테이션도 세팅돼 있습니다. LLM은 기본 ASK_URL=https://askbot.ce.moreh.dev/ask(사내 게이트웨이) — Anthropic 직통을 쓰려면 Secret의 ANTHROPIC_API_KEY만 채우고 Deployment의 ASK_URL env를 지우면 됩니다.

Secret에 실제 값을 넣은 파일은 커밋하지 마세요. 아래처럼 kubectl로 직접 만들거나 sealed-secrets 사용.
# 시크릿만 따로 만드는 방법(매니페스트의 Secret 블록 대신)
kubectl -n «NAMESPACE» create secret generic repolens-secrets \
  --from-literal=GH_CLIENT_ID=«id» \
  --from-literal=GH_CLIENT_SECRET=«secret» \
  --from-literal=ANTHROPIC_API_KEY=«key»   # ASK_URL만 쓸 거면 생략

3 적용 CLI

kubectl apply -f deploy/k8s/repolens.yaml
kubectl -n «NAMESPACE» rollout status deploy/repolens

또는 gitops면 deploy/k8s/README.md 의 Argo CD Application 을 추가(자동 sync).

4 확인 CLI + 웹

kubectl -n «NAMESPACE» get pods -l app=repolens
kubectl -n «NAMESPACE» get ingress repolens          # ADDRESS/HOSTS 확인
# 클러스터 내부 헬스
kubectl -n «NAMESPACE» port-forward deploy/repolens 8080:8080 & curl -s localhost:8080/healthz   # → ok
# 인증서 발급 대기(cert-manager)
kubectl -n «NAMESPACE» get certificate repolens-tls

웹 인증서가 Ready 되면 https://repolens.ce.moreh.dev 열기 → Sign in with GitHub → 레포 Load → 트리·전문검색·심볼 그래프·GraphRAG.

5 (선택) 공개 데모도 이 백엔드로 CLI

han-oqo.github.io/repo-lens 가 이 백엔드를 쓰게 하려면 repo 변수만:

gh auth switch --user HAN-oQo
gh variable set REPOLENS_API_BASE   --repo HAN-oQo/repo-lens --body https://repolens.ce.moreh.dev
gh variable set REPOLENS_OAUTH_BASE --repo HAN-oQo/repo-lens --body https://repolens.ce.moreh.dev
gh workflow run "Deploy to GitHub Pages" --repo HAN-oQo/repo-lens
gh auth switch --user hanq-moreh

그리고 OAuth App 콜백/ALLOWED_REDIRECTS에 https://han-oqo.github.io/repo-lens 도 추가하면 공개 데모에서 로그인→v2 동작.

6 업데이트 · 롤백 CLI
